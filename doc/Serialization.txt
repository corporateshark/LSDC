Теперь поддерживается автогенерация кода сериализации.

Все процедуры SaveToXLML/AcceptParameter сохраняются для каждого класса в указанный файл.
Для всех Package'ей можно указывать SerializationList - файл со списком классов, для
которых необходимо сгенерировать сериализационный код.

Формат файла
  <ClassName>  <OutFile>


Т.е., если мы для Core хотим сохранять классы LProjector и LMaterial, а также
желаем, чтобы процедуры AcceptParameter/SaveToXLML сохранились бы в файл CoreSerialization.cpp,
то пишем в Serialize.list:

; начинаем
LProjector   CoreSerialization.cpp
; продолжаем
LMaterial
; для LMaterial файла не указано - он берётся из последнего указанного

(допускаются комментарии, начинающиеся с ';')

Для Package'а _Core_ в LSDC.cfg пишем строчку
SerializationList = Serialize.list

==========

При сериализации предполагается, что все сериализуемые поля имеют конвертеры в/из строки.
Конвертеры указываются в отдельном файле, имя файла указывается в LSDC.cfg
TypeConvertersList=<имя>  (по умолчанию - TypeConverters.list).

Формат списка преобразователей:

<имя native-типа>  <преобразователь в LString>    <преобразователь из LString>

В файле TypeConverters допускаются комментарии
